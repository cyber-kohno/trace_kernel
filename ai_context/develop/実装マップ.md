# Trace Kernel 実装マップ

## 目的

`ai_context/spec` の仕様に対して、実装コード上の対応箇所を追いやすくするためのマップ。

- フロントエンド
  - `app/src`
  - Svelte / TypeScript
- バックエンド
  - `app/src-tauri/src`
  - Rust / Tauri command


## 全体アーキテクチャ

Trace Kernel は大きく以下の 4 層で構成されている。

1. 画面レイヤ
   - スタート画面、ワークスペース管理画面、プログラムエディタ、トランザクションダイアログ
2. 状態管理レイヤ
   - ワークスペース、選択中要素、enable/error 状態、dirty 状態、キャッシュ
3. スクリプト実行レイヤ
   - Monaco による型注入
   - ブラウザ Worker 上での TypeScript 実行
   - `$env/$resource/$dataset/$process` と独自 API の注入
4. ネイティブ処理レイヤ
   - Tauri invoke 経由で Rust 側のファイル操作、スキャン、外部プロセス実行、ストリーム保持

## 仕様要素ごとの対応

### env

#### 仕様上の役割

- 環境変数的なキーと値の管理
- パス用途チェック
- `%NAME%` 形式で他要素から参照

#### 実装箇所

- 状態
  - `app/src/app/store/storeEnv.ts`
- 詳細画面
  - `app/src/app/contents/maintenance/MDEnv.svelte`
- パス存在チェック UI
  - `app/src/app/util/form/validation/PathState.svelte`
- 参照展開
  - `app/src/app/util/data/dataUtil.ts`
  - `app/src/app/contents/detail/program/util/contextDataUtil.ts`

#### 補足

- 変数名は大文字のみ
- 実行時には `$env.NAME` として注入


### resource

#### 仕様上の役割

- 静的テキスト資源の割り当て
- CSV / TSV のパース
- 型付きで `$resource.xxx` に注入

#### 実装箇所

- 状態
  - `app/src/app/store/StoreResource.ts`
- 詳細画面
  - `app/src/app/contents/maintenance/MDResourece.svelte`
- CSV / TSV 変換
  - `app/src/app/util/data/dataUtil.ts`
- 型宣言生成
  - `app/src/app/contents/detail/program/util/contextDataUtil.ts`

#### 補足

- `parse` 未指定なら `string`
- `parse` 指定時はオブジェクト配列として注入


### dataset

#### 仕様上の役割

- ファイル群の動的リソース管理
- ルートパス、エンコード、スキャン条件、選択対象を保持
- `Runtime auto` または `Direct choose`

#### 実装箇所

- 状態
  - `app/src/app/store/storeDataset.ts`
- 詳細画面親
  - `app/src/app/contents/maintenance/dataset/MDDataset.svelte`
- スキャン条件画面
  - `app/src/app/contents/maintenance/dataset/scan/ScanCondPhase.svelte`
- 選択画面
  - `app/src/app/contents/maintenance/dataset/choose/ChoosePhase.svelte`
- 選択補助
  - `app/src/app/contents/maintenance/dataset/choose/chooseUtil.ts`
- 一覧画面
  - `app/src/app/contents/maintenance/dataset/ListPhase.svelte`
- scan 結果キャッシュ
  - `app/src/app/store/storeCache.ts`
- Rust スキャン本体
  - `app/src-tauri/src/scan.rs`

#### 補足

- `targets == null`
  - `Runtime auto`
  - 実行前に Rust 側で走査
- `targets != null`
  - `Direct choose`
  - GUI で確定した相対パス群を使用
- 実行時には `$dataset.xxx[]` として注入され、各要素は以下を持つ
  - `fileName`
  - `absolutePath`
  - `relativePath`
  - `content(): Promise<string>`


### process

#### 仕様上の役割

- 外部プログラムを関数として呼び出す
- スクリプト引数をコマンドライン引数へ注入
- stdout / stderr / exitCode を返す

#### 実装箇所

- 状態
  - `app/src/app/store/StoreProcess.ts`
- 詳細画面
  - `app/src/app/contents/maintenance/MDProcess.svelte`
- 実行時の注入
  - `app/src/app/contents/detail/program/util/contextDataUtil.ts`
- Rust 側の実行本体
  - `app/src-tauri/src/process.rs`

#### 補足

- 実行時には `$process.funcName(...)` として注入
- `__argName__` をコマンドライン引数文字列に埋め込む
- unresolved なスクリプト引数が残るとフロント側でエラー
- Rust 側では timeout 管理と stdout のバイナリ検出を実施
- 実装上は Professional ライセンス向け機能


### work

#### 仕様上の役割

- TypeScript を書いて実行する単位
- `plain` / `channel` の出力方式を持つ

#### 実装箇所

- 状態

  - `app/src/app/contents/maintenance/work/MDWork.svelte`

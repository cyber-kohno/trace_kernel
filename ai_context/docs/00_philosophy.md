# Philosophy — Programming as a Tool

## 背景

多くのエンジニアにとって、プログラムは「アプリを作るためのもの」です。

そのため、以下のような日常的な問題に対して、わざわざ「プログラムを書こう」という発想になりません。

- ログを分析する
- データを加工する
- ファイルを整理する

書くことが難しいのではありません。  
**実行できる状態にするまでが重い**のです。

## プログラミングを「特別な行為」にするもの

プログラムを実行するまでには、以下の準備が必要です。

- 環境構築
- 依存関係の解決
- プロジェクトの作成
- コンパイル・トランスパイル設定
- ターミナルでの実行
- ファイルを読み込むためのボイラープレートコード

この準備コストが、プログラミングを日常の道具から遠ざけています。

## 本来あるべき姿

電卓を使うとき、環境構築はしません。  
メモを書くとき、プロジェクトは作りません。

**道具とは、本来そういうものです。**

もし、TypeScriptが、アプリを開いた瞬間に書いて、その場で実行できるとしたら。  
必要なリソースが整理され、すぐに参照することができるとしたら。

プログラミングは、もっと身近な道具になるはずです。

## Trace Kernelの立場

> プログラムは、開発のための手段にとどまらない。  
> それは本来、問題を解くための道具である。

Trace Kernelは、その当たり前を取り戻すために設計された実行基盤です。

TypeScriptを、環境構築不要で即書いて即実行できる状態にすること。  
必要なリソースをGUIで管理し、プログラムに自動注入すること。  

これにより、エンジニアはビジネスロジックのみに集中できます。

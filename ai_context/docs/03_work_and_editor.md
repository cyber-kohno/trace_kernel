# work とプログラムエディタ

## work の概要

`work` は、TypeScriptのプログラムを書いて実行する**作業単位**です。

- プログラムのファイル分割・インポート・エクスポートはなく、**1ワーク = 1プログラム**
- ワーク間の自動連携はなく、**完全に独立**している
- 複数のworkを使って処理をフェーズ分けし、手動で順に実行していくことを想定

### フェーズ分けの例

```
work1: データの収集 → 出力結果をコピーしてリソースに手動で貼り付け
work2: データの解析・整形 → 同様に中間データをリソースへ保存
work3: 最終的なファイル出力
```

## work の設定項目

| 項目 | 説明 |
|------|------|
| `name` | ワーク名（識別用。他から参照されない） |
| `output_method` | 出力方式 |
| `api_injections` | 注入される独自API（output_methodにより変わる） |
| `context_injections` | 注入可能なコンテキスト一覧（errorのものは表示されない） |

## output_method

output_method により、注入される出力系APIが変わります。

| 選択肢 | 注入されるAPI | 説明 |
|--------|-------------|------|
| `Plain` | `$print` / `$println` | シンプルなテキスト出力。基本的にはこちらを使用。 |
| `Channel` | `$channel` | 複数ストリームへの出力に対応。テキスト・テーブル形式で出力できる。 |

## プログラムエディタ

workの詳細画面で「Open Editor」ボタンを押下すると、プログラムエディタが全画面で表示されます。

### エディタの特徴

- **Monaco Editor**（VS Code と同等のエディタ）を採用
- ワークスペースで定義したコンテキストが**補完情報として自動注入**される
- 編集内容はリアルタイムでワークスペースに反映（保存・取り消し操作なし）
- `×` ボタンまたは `ESC` キーでエディタを閉じる

### 補完の例

```typescript
$resource.userList[0].id   // CSVのカラム名が補完される
$env.DEST_DIR               // envで定義した変数名が補完される
$dataset.workspace          // datasetで定義した変数名が補完される
```

## プログラムの実行

### 実行方法

以下のいずれかの操作でプログラムを実行します。

| 操作 | 方法 |
|------|------|
| ボタン | 右下の「Run」ボタン |
| ショートカット | `F5` または `Alt+Enter` |

### 実行中の画面

実行を開始すると、エディタが以下のレイアウトに切り替わります。

```
左半分：ソースコード（表示のみ）
右半分：出力結果パネル
```

- 実行中は「Executing...」と表示される
- `$state` APIを使うことでプログレスバーやモニターを追加表示できる
- 「Clipboard」ボタンで出力結果をクリップボードにコピー可能

### 実行のキャンセル

| 操作 | 方法 |
|------|------|
| ボタン | 画面下部の「Cancel」ボタン |
| ショートカット | `Alt+左キー` |

キャンセルするとソースコード編集画面に戻ります。

## プログラム内で使えるもの

workのプログラム内では、`$`（ドルマーク）で始まる要素が自動注入されます。

### コンテキスト（ワークスペースで定義したもの）

```typescript
$env.変数名
$resource.変数名
$dataset.変数名
$process.関数名()
```

### 独自API（Trace Kernelが提供するもの）

| API | 役割 |
|-----|------|
| `$print` / `$println` | GUI出力パネルへの文字列出力（Plain時） |
| `$channel` | マルチストリーム出力（Channel時） |
| `$state` | プログレスバー・モニターの表示 |
| `$fs` | ファイルシステムの読み書き |
| `$parse` | Excel / XML / CSV などのパース |
| `$net` | HTMLの取得・外部APIの呼び出し |

詳細は各APIのリファレンスドキュメントを参照してください。
